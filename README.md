# 📊 Telecom Analysis — End-to-End Data Analysis (EDA)

## Project Objective
This project aims to analyze customer behavior in a telecommunications company to understand usage patterns, identify customer segments, detect outliers, and generate actionable business insights to optimize service plans and improve customer experience.

---

## Datasets Used
The analysis is based on three datasets:

- **plans.csv**  
  Contains information about available plans (pricing, included minutes, data, extra costs).

- **users_latam.csv**  
  Customer data including age, city, registration date, plan, and churn status.

- **usage.csv**  
  Detailed user activity including calls (duration) and messages (length).

## 🔄 Analysis Workflow
The project follows an end-to-end data analysis process:

1. **Step 1: Load & Explore**
   - 1.1 Data loading and quick view
   - 1.2 Exploring the structure of the datasets

2. **Step 2: Identifying Data Quality Problems**
   - 2.1 Review of Null Values
   - 2.2 Detection of invalid values ​​and sentinels
   - 2.3 Review and standardization of dates

3. **Step 3: Basic Data Cleaning**
   - 3.1 Correct sentinels and impossible dates
   - 3.2 Correct sentinels and impossible dates

4. **Step 4: Summary usage statistics per user**
     - 4.1 Grouping by usage behavior
     - 4.2 Statistical summary by user during 2024
     - Total call minutes
   - User segmentation (age groups and usage levels)

5. **Step 5: Visualization of distributions (usage and customers) and outliers**
   - 5.1 Visualization of Distributions
   - 5.2 Identification of Outliers

6. **Step 6: Customer Segmentation**
   - 6.1 Customer Segmentation by Usage
   - 6.2 Customer Segmentation by Age
   - 6.3 Visualization of Customer Segmentation

7. **Step 7: Executive Insight for Stakeholders**
   - Executive analysis:
     - Problems detected in the data
     - Age Segments
     - Segments by Usage Level
     - Recommendations
